Part I

The Elements of Style

What is CSS?

  • CSS stands for cascading style sheets
  • CSS is a declarative language that describes how webpages should be styled
  • Styles in CSS are prioritized in a particular order (hence "cascading")
  • Today, HTML, CSS, and Javascript form the basis of front-end web development

Specificity: Add It Up

  • IDs (#about)
  • Classes, pseudo-classes (.row, :hover)
  • Elements, pseudo-elements (a, ::before)
            <div id="about">
              <a class="row" href="tel:5558675309">Phone: (555) 867-5309</a>
              <a class="row" href="mailto:noreply@example.com">Email: noreply@example.com</a>
            </div>
          
            #about a.row:hover::before {
              /* specificity 1-2-2 */
            }
            a:hover {
              /* specificity 0-1-1 */
            }

The Box Model

  • Every layout engine is still built around the box model
  • Basic concepts established in CSS1
  • It makes sense considering what CSS was designed for

Flow

  • Elements display as either block or inline (and sometimes inline-block)
  • Inline and inline-block elements flow top to bottom, left to right
  • Block elements only flow top to bottom, unless it has float
  • This can be overridden by the display property
  • position: absolute; and position: fixed; take elements out of flow entirely

Layering

  • z-index only works with set position—a new stacking context
  • z-index orders within the context
  • Alternatives: float and negative margin
